# Доменный глоссарий

> В коде используем **английские эквиваленты** (правая колонка) — это про
> идентификаторы (поля моделей, имена компонентов, ключи API), а не про
> переводы UI. UI на v1 — **только русский**, без `@nuxtjs/i18n` и переводов
> на uz/en (см. корневой `CLAUDE.md`, раздел «Локализация»).

| RU (UI / переговоры) | EN (код, API, БД) | Описание |
|---|---|---|
| Мероприятие | `event` | Конференция / выступление / любая активность с билетами. |
| Категория билета | `ticketTier` / `tier` | Тип билета внутри мероприятия (Standard / VIP / Online…). Содержит цену и описание. |
| Билет | `ticket` | Конкретный экземпляр купленного билета (с анкетой и QR). |
| Заказ | `order` | Покупка одного или нескольких билетов в одной транзакции. |
| Позиция заказа | `orderItem` | Связь Заказ ↔ TicketTier с количеством. |
| Спикер | `speaker` | Карточка человека (ФИО, фото, профиль), привязывается к мероприятию. |
| Организатор | `organizer` | Кто проводит мероприятие. (Уточнить: отдельная сущность или User-роль.) |
| Программа мероприятия | `agenda` / `program` | Расписание/тайминги мероприятия. |
| Анкета посетителя | `attendeeForm` | Поля, которые посетитель заполняет на каждый билет в checkout (ФИО, контакты, доп.). |
| Посетитель | `attendee` | Человек, на которого выписан билет (ФИО из анкеты). |
| Промокод | `promoCode` | Купон / скидка. |
| «Мне интересно» / Избранное | `favorite` / `wishlist` | Закладка пользователя на мероприятие. |
| Способ оплаты | `paymentMethod` | `card` \| `sbp` \| `invoice`. |
| Картой | `card` | Оплата банковской картой. |
| СБП | `sbp` | Система быстрых платежей. |
| По счёту | `invoice` | Безнал / B2B-счёт. |
| Согласие на ПД | `personalDataConsent` | Чекбокс согласия в checkout. |
| Главная | `home` | Лента ближайших мероприятий. |
| Поиск | `search` | Экран поиска по названию. |
| Купленные билеты | `myTickets` | Список купленных. |
| Личный кабинет (ЛК) | `account` | Профиль пользователя. |
| Связаться с менеджером | `contactManager` | Пункт меню в ЛК. |
| Стать спикером | `becomeSpeaker` | Заявка на роль спикера. |
| Выйти из аккаунта | `signOut` | Logout. |
| QR-код билета | `ticketQr` | Уникальный код для прохода. |
| Номер заказа / билета | `orderNumber` / `ticketNumber` | Идентификаторы для пользователя. |
| Сессия устройства | `deviceSession` | Уже реализовано: `apps/backend/src/api/session/`. |

## Соглашения о коде

- Поля БД и API — `camelCase` (Strapi 5 default).
- Идентификаторы внешних API (платёжный шлюз, СБП) хранить отдельным полем
  с префиксом провайдера: `stripePaymentIntentId`, `sbpQrId` и т. п.
- Цены — целые числа в **минимальных единицах валюты** (копейки) + поле
  `currency` (на v1 только `RUB`; enum оставлен расширяемым).
- Дата/время — UTC в БД (ISO 8601 с `Z`); таймзона мероприятия — отдельным
  полем `timezone` (IANA, на старте `Europe/Moscow`).
