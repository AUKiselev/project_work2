# UI-спецификация v1 (по вайрфреймам)

> Источник: `Приложение, совет такси.pdf` (название файла нерелевантно). Здесь —
> текстовая транскрипция всех экранов. Используй этот документ при создании
> страниц/компонентов и обсуждении продукта. Любое расхождение между этим документом
> и реальностью — ставить вопрос продукт-менеджеру, не «улучшать» молча.

## Глобальные элементы

### Bottom navigation (на всех основных экранах)

Четыре пункта (слева направо):

1. **Главная** — лента мероприятий «в порядке отдаления от даты просмотра»
   (т. е. ближайшие сверху).
2. **Поиск** — поиск мероприятия по названию.
3. **Купленные билеты** — список билетов, которые купил пользователь.
4. **ЛК** (Личный кабинет) — фото, ФИО, привязанные карты, настройки.

### Карточка мероприятия (повторно используемый компонент)

Применяется на главной, в поиске, в купленных билетах (с вариациями):

- Фото мероприятия (главное).
- Даты проведения.
- Название мероприятия.
- Краткое описание мероприятия.
- Бейдж/кнопка **«Мне интересно»** (избранное / wishlist).
- **«Цена от»** (минимальная цена среди категорий билетов).

В купленных билетах вариант проще: фото + название + дата и место (без цены и
«мне интересно»).

## Экран 1. Главная

Сверху вниз:

1. **Logo** в правом верхнем углу.
2. **Рекламный баннер-слайдер** под наши мероприятия.
3. **Лента карточек мероприятий** (см. «Карточка мероприятия» выше).
   Сортировка — ближайшие сверху, дальше — ниже.
4. **Bottom navigation** (см. выше).

## Экран 2. Поиск

1. **Иконка лупы + поле «Поиск»** сверху (поиск по названию мероприятия).
2. **Лента результатов** в формате компактной карточки: фото слева, справа
   название мероприятия + дата и место проведения.
3. **Bottom navigation**.

## Экран 3. Детальная страница мероприятия

Шапка:

- Кнопка **«Назад»** (слева).
- Кнопка **«Мне интересно»** (справа).
- Кнопка **«Поделиться»** (справа).

Содержимое:

1. **Фото мероприятия** (главное / hero).
2. **Название мероприятия** (большой заголовок).
3. **Блок «Кто проводит»**: фото + название организатора + кнопка
   **«ещё мероприятия»** (другие мероприятия этого организатора).
4. **Дата проведения** (кратко).
5. **Место проведения** (кратко).
6. **Описание мероприятия** (текст).
7. **Программа мероприятия** (расписание / тайминги).
8. **Карта** с адресом мероприятия.
9. **Блок со спикерами**:
   - Горизонтальный список карточек спикеров.
   - По клику на спикера → его профиль.
10. **Фото с прошлого мероприятия** (галерея).
11. Подвал:
    - Слева: **Цена от и до** (диапазон).
    - Справа: **Кнопка «Купить»** → переход на экран выбора билетов.

## Экран 4. Выбор и покупка билетов (cart)

Шапка:

- Кнопка **«Назад»**.
- Кратко: название мероприятия, дата, место.

Содержимое — повторяющийся блок на каждую категорию билета (Билет 1 / 2 / 3 / …):

1. Заголовок: **«Билет N» + краткое описание**.
2. **Кнопка выбора количества билетов** в этой категории (степпер `−` `N` `+`).
3. **Кнопка-аккордеон** (сноска в вайрфрейме: «Кнопка позволяет развернуть
   наполнение билета, чтобы понять, что в него входит») — раскрывает детали
   категории.
4. Справа: **Стоимость всех выбранных билетов этой категории**.

Подвал:

- Слева: **Итоговая стоимость всех билетов** + рядом кнопка
  **«Детализация билетов»** (раскрытие списка).
- Справа: **поле «Промокод»**.
- Внизу по центру: **Кнопка «Купить»** → переход на экран оформления.

## Экран 5. Оформление заказа (checkout)

Шапка:

- Кнопка **«Назад»**.
- Картинка мероприятия (миниатюра).
- Кратко: название мероприятия, дата, место.
- **Стоимость конкретно этого (выбранного) билета** или итоговая.

Содержимое:

1. **Анкета первого билета** — поля посетителя (ФИО + контакты + любые
   доп. поля, заданные организатором).
2. **Анкета второго билета** — то же, для следующего.
3. … (по числу билетов в заказе).
4. **Способ оплаты** — radio-выбор:
   - **Картой**.
   - **СБП**.
   - **По счёту** (B2B / счёт на оплату).
5. **Дата и время + Место** (повтор для уверенности).
6. Справа от подвала: **Итоговая сумма**.
7. **Чекбокс «Согласие на обработку персональных данных»**.
8. Кнопка подтверждения покупки (в спеке не подписана явно — подразумевается
   «Оплатить» / «Подтвердить»).

## Экран 6. Купленные билеты (список)

1. **Лента карточек**: фото слева + справа название мероприятия и дата/место.
2. **Bottom navigation**.

По клику на карточку — экран 7.

## Экран 7. Билет (детально, для прохода)

Шапка:

- Справа сверху: **«Скачать»**, **«Поделиться»** (кнопки).

Содержимое:

1. **QR-код** билета (или общий QR заказа).
2. **Название мероприятия**.
3. **Даты и время проведения**.
4. **Название билета (категория) + стоимость**.
5. Справа под шапкой: **Номер заказа / Номер билета / Заказчик**.
6. **Карта** с адресом мероприятия.
7. **Информация об организаторах**.
8. **Правила использования билета**.
9. Внизу — стрелка вверх (вероятно sticky-кнопка «к началу» / «свернуть»).

### Подэкран: список посетителей заказа

Если в одном заказе несколько билетов:

- Кнопка **«Назад»**.
- Список строк: **ФИО посетителя** + кнопки **«Поделиться»** / **«Скачать»** + стрелка
  вниз (раскрывает QR конкретного посетителя).

## Экран 8. Личный кабинет (ЛК)

1. **Фото пользователя** (аватар) + **ФИО**.
2. Пункты меню:
   - **Избранное** — мероприятия, отмеченные «мне интересно».
   - **Связаться с менеджером**.
   - **Стать спикером** (форма заявки).
3. Внизу: **Выйти из аккаунта**.
4. **Bottom navigation**.

Дополнительно из README текущей реализации: уже есть страница
`/sessions` (управление активными устройствами) — её надо добавить пунктом меню в ЛК.

## Открытые вопросы / TBD

Это не описано в вайрфреймах и требует уточнения у заказчика:

- Аутентификация на старте: гость или сразу логин? (текущий код предполагает логин.)
- Рекламный баннер-слайдер — кем модерируется, какие поля?
- «Кто проводит» — отдельная сущность Organizer или это связь с User?
- Привязка карт в ЛК — какой провайдер (ЮKassa / Stripe / другой)?
- Пуш-уведомления (напоминание о мероприятии, статусы заказа) — да / нет?
- Возврат билета и политика возвратов.
- Многоязычность контента: на v1 — RU only (зафиксировано). Когда понадобится
  uz/en — отдельная задача.
- Capacity / лимит билетов в категории — есть или нет?
